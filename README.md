# official-main-guide

本仓库用于归档和发布多个独立的 HTML 页面。这里不针对任何特定域名或网站，仅作为静态页面的集中存放与版本管理。

## 项目简介

这是一个通用型 HTML 页面归档仓库。您可以将需要独立存放、展示或分发的 HTML 文件提交至此，便于统一管理和版本追踪。仓库本身不提供任何动态服务，所有页面均为纯前端静态内容。

## 目录说明

```
/
├── index.html          # 仓库首页（可选）
├── pages/              # 主要页面存放目录
│   ├── example1.html
│   └── example2.html
├── assets/             # 公共资源（CSS、JS、图片等）
│   ├── css/
│   ├── js/
│   └── images/
└── README.md           # 本文件
```

- `pages/`：存放所有独立 HTML 页面，建议按功能或主题命名。
- `assets/`：页面引用的公共静态资源，可按类型进一步细分。

## 页面归档说明

每个 HTML 页面应保持独立完整，尽可能自包含（CSS/JS 内联或引用本仓库资源）。提交新页面时，请遵循以下约定：

- 页面文件名使用小写字母、数字和连字符（如 `user-guide.html`）。
- 在页面 `<head>` 中注明标题和简单的描述（meta description）。
- 如需引用外部资源，优先使用本仓库 `assets/` 下的文件。

## 维护说明

- 本仓库接受 Pull Request 以新增或更新页面。
- 请确保提交的 HTML 页面无恶意代码、无外部追踪脚本。
- 删除或修改已有页面时，请在 Commit 信息中说明理由。
- 建议定期检查页面链接与资源引用是否有效。

## 使用方式

您可以直接通过 GitHub Pages 或其他静态托管服务部署本仓库，从而在线访问各 HTML 页面。部署后，每个页面将拥有独立的 URL。

## 贡献

欢迎提交 Issue 或 Pull Request。所有贡献请保持简洁、清晰，并遵循本仓库的基本规范。
